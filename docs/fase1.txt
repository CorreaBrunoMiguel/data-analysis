# Fase 1 – Design do Produto e UX/UI | Checklist de Sprint

**Objetivo:** Planejar a interface, jornada do usuário e experiências gamificadas, garantindo consistência, usabilidade e foco no aprendizado interativo.

---

## 1. Mapeamento da Jornada do Usuário

* [ ] Identificar todos os pontos de interação do usuário: onboarding → módulos → quizzes → desafios → exercícios → projetos → conclusão
* [ ] Definir caminhos alternativos e fluxos de decisão (usuário pode escolher ordem de quizzes/desafios/exercícios)
* [ ] Mapear checkpoints e feedbacks automáticos em cada etapa
* [ ] Criar diagrama de fluxo do usuário detalhado, incluindo tela de escolha pós-teoria e fluxo de XP
* [ ] Detalhar pontos de engajamento e interação gamificada

**Entregável:** Mapa completo da jornada do usuário com fluxos de decisão e checkpoints

---

## 2. Definição de Telas e Componentes de UI

* [ ] Listar todas as telas necessárias: dashboard, módulo, subtópico, quiz, desafio, exercícios, resultados, tela de escolha pós-teoria, ranking, badges
* [ ] Definir componentes visuais: botões, campos de pontuação, barra de progresso, badges, XP, mensagens de feedback, alertas de conquistas
* [ ] Definir padrão de cores, tipografia, ícones e layout (mantendo consistência, clareza e acessibilidade)
* [ ] Documentar comportamento interativo de cada componente (hover, clique, mensagens, atualizações de XP)

**Entregável:** Documento de telas e componentes, padronização visual

---

## 3. Criação de Wireframes e Mockups

* [ ] Criar wireframes de baixa fidelidade para cada tela principal, incluindo telas de teoria, quiz, desafio e exercícios
* [ ] Criar mockups de média/alta fidelidade incorporando identidade visual, cores, badges e barra de progresso
* [ ] Incluir representação visual de XP acumulado, pontuação, badges desbloqueadas e feedback do usuário
* [ ] Garantir que a navegação entre telas siga o fluxo mapeado

**Entregável:** Wireframes e mockups aprovados para prototipagem interativa

---

## 4. Documentação de Regras de Gamificação

* [ ] Documentar regras detalhadas de XP e badges (conforme Fase 0)
* [ ] Definir como cada tipo de conteúdo contribui para XP: teoria (checkpoints sem XP), quizzes (0-100 XP), desafios (0-100 XP), exercícios (0-100 XP)
* [ ] Documentar conquistas especiais, desbloqueio de mundos/tópicos e ranking
* [ ] Registrar pontos de integração com o fluxo de usuário, tela de escolha e feedback visual
* [ ] Definir limites de tentativas e regras de pontuação para quizzes, desafios e exercícios

**Entregável:** Documento atualizado de gamificação e XP, pronto para implementação

---

## 5. Validação do Protótipo de Fluxo de Interação

* [ ] Validar protótipo com foco no usuário final (navegação, clareza, interatividade)
* [ ] Ajustar elementos de UX/UI com base em testes internos ou feedback de usuários simulados
* [ ] Garantir consistência com regras de gamificação e fluxo de aprendizado, incluindo feedback de XP e pontuação
* [ ] Documentar alterações e aprovações de protótipo para referência futura

**Entregável:** Protótipo validado e documentado, pronto para desenvolvimento dos módulos interativos

---

**Prioridade:** Alta
**Duração estimada:** 2 semanas
**Observação:** Pode ser menor se o design for feito de forma iterativa e rápida. Inclui mapeamento completo de tela de escolha, XP, badges, pontuação e feedback visual.
